# 附录 — 面试话术速查卡

> **使用方式**：面试前 30 分钟快速过一遍，面试时在脑中作为"索引"使用。

---

## 一、30 秒自我介绍模板

"我做过一套完整的 Agent 全栈平台，7 个项目从底层 Token 引擎到上层评测平台到容器部署全覆盖。技术栈是 Rust + Python + TypeScript + Docker + K8s。最核心的项目是 AgentForge——一个可插拔的 Agent 集成框架，中间件管道架构，支持 LangChain、OpenAI Agents SDK 等 3 种框架适配，有智能路由自动选择最经济的模型。所有项目都有 docker compose 一键启动和完整的自动化测试。"

---

## 二、万能回答框架（STAR-L）

面试必用结构：
- **S**ituation（场景）：我们的 Agent 开发流程中遇到了什么问题
- **T**ask（任务）：我需要解决什么
- **A**ction（行动）：我做了什么（技术选型 + 架构设计 + 实现）
- **R**esult（结果）：达到了什么效果（数据化）
- **L**earning（反思）：如果重来我会怎么做

## 四、最容易加分的 5 个回答

1. **"我们做了缓存机制"** → 加分，但更好的是 **"缓存命中率低时，系统会自动建议启用和配置缓存"**
2. **"服务挂了会重试"** → 加分，但更好的是 **"重试用了指数退避（1s/2s/4s/8s），避免打垮下游"**
3. **"我们做了安全防护"** → 加分，但更好的是 **"纵深防御：输入校验→权限检查→沙箱隔离→审计追溯"**
4. **"速度很快"** → 加分，但更好的是 **"1000 条消息批量 Token 计数 <10 毫秒，Python 实现需要 50 毫秒"**
5. **"代码质量好"** → 加分，但更好的是 **"CI 门禁：lint + test + security scan，任何一步失败阻断合并"**

---

## 五、最容易扣分的 5 个回答

1. **"这个没遇到问题，很顺利"** → 面试官听到的是"你没深入做"
2. **"我就是照着文档写的"** → 面试官听到的是"你没自己的理解"
3. **"X 比 Y 好所以用 X"** → 面试官听到的是"你不会做技术选型"
4. **"测试覆盖率 50%"** → 面试官听到的是"你只测了 happy path"
5. **"我负责的是前端部分"**（在问全栈项目时） → 面试官听到的是"你不懂后端"

---

## 六、面试官沉默了怎么办

面试官沉默 = 在等你补充更多内容。这时候是最佳的**加分时刻**：

- "如果让我展开讲，这个设计的核心考量是..."
- "一个反直觉的地方是...（讲一个出人意料的点）"
- "最大的坑是...（讲一个你踩过并解决的坑）"
- "如果重新做，我会...（展示你的反思能力）"

---

## 七、面试结束时的反提问

面试官问"你有什么想问我的吗？"——这是最后的机会展示你的思考深度。

**好问题**（展示你对工程的理解）：
- "团队目前是怎么做 Agent 评测的？"
- "代码审查的流程是什么样的？merge 之前要过几个 approve？"
- "服务的 on-call 机制是怎样的？多久轮一次？"

**坏问题**（可以用 Google 搜到的）：
- "公司用什么技术栈？"
- "公司有多少人？"

---

## 八、心态最后一句话

**面试不是考试，是对话。面试官不是在打分，是在找同事。你的目标不是"答对所有问题"，而是"让对方觉得你是一个可以一起解决问题的人"。**

深呼吸，放轻松，你准备得很充分了。
